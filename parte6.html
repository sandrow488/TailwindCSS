How does this even work?

// Button.jsx
export default function Button({ size, children }) {
  let sizeClasses = {
    md: "px-4 py-2 rounded-md text-base",
    lg: "px-5 py-3 rounded-lg text-lg",
  }[size];
  return (
    <button type="button" className={`font-bold ${sizeClasses}`}>
      {children}
    </button>
  );
}

// Using CSS vars + utilities
export function BrandedButton({ buttonColor, buttonColorHover, textColor, children }) {
  return (
    <button
      style={{
        "--bg-color": buttonColor,
        "--bg-color-hover": buttonColorHover,
        "--text-color": textColor,
      }}
      className="bg-[--bg-color] text-[--text-color] hover:bg-[--bg-color-hover] rounded-md px-3 py-1.5 font-medium"
    >
      {children}
    </button>
  );
}

Ejercicio guiado — Componente de ficha de producto solo con utilidades

Construye una ficha de producto 100% con utilidades de Tailwind (sin CSS adicional) que sea responsiva, accesible y coherente con el sistema de diseño. Debe incluir imagen o placeholder, badges de estado, título, descripción breve y dos llamadas a la acción. El layout debe adaptarse en función del ancho disponible y mantener una jerarquía visual clara.

Requisitos mínimos

  • Diseño en una o dos columnas usando utilidades de grid o flex.
  • Placeholder de imagen cuadrado con degradado o patrón simple.
  • Badges con estados diferenciados (por ejemplo, categoría y disponibilidad).
  • Jerarquía tipográfica clara con tamaños como text-xs, text-sm, text-2xl.
  • Dos CTAs: primario sólido y secundario con borde o estilo neutro.
  • Compatibilidad con modo oscuro usando dark:.
  • Accesibilidad: roles/labels adecuados y foco visible con focus:.

Pasos guiados

  • Envuelve el componente en un contenedor con tarjeta: borde, radio, sombra ligera y padding.
  • Crea el layout base en una columna y añade una variante responsiva a dos columnas usando grid con gap. Un patrón útil es md:grid-cols-[auto,1fr].
  • Genera un placeholder de imagen cuadrado con size-24 o similar y un degradado con bg-gradient-to-br aprovechando from-brand/20.
  • Añade una línea de badges: uno para la categoría y otro para el estado del producto (por ejemplo, “Nuevo” o “En stock”). Usa utilidades de fondo, borde y tipografía.
  • Escribe un título con text-2xl y un párrafo con text-sm para descripción breve.
  • Coloca dos botones: el primario con btn btn-brand y el secundario con borde y texto neutro. Añade focus:outline-none y focus:ring-2 con un color coherente.
  • Revisa el contraste en claro/oscuro y ajusta con utilidades dark:.

Retos unas vez resuelta la base

  • Añade un precio con text-xl font-bold y un badge “-20 %” en la esquina del placeholder usando relative y absolute.
  • Incluye un pequeño bloque de beneficios en lista con list-disc y pl-4.
  • Usa @container en el artículo para cambiar a dos columnas solo cuando el contenedor lo permita, por ejemplo class="@container md:@container grid @md:grid-cols-[auto,1fr]" si estás trabajando con consultas por contenedor.
  • Prueba variantes de estado en los botones: hover:, active:, focus-visible:, y disabled: con aria-disabled.

Checklist

  • ¿Se lee bien el título y hay contraste suficiente en claro/oscuro?
  • ¿El layout pasa a dos columnas de forma fluida en pantallas medianas?
  • ¿Los botones muestran foco visible con focus:ring?
  • ¿No has escrito CSS manual, solo utilidades?
